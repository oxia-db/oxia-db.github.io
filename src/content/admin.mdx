# Admin Operations

Oxia provides an admin interface for cluster management operations. The admin service runs on the
coordinator and is accessible via the `oxia admin` CLI or the admin gRPC API.

The admin gRPC API is now exposed as a draft reference in
[Admin API](./reference/admin-api.mdx).

## Connecting to the admin service

The coordinator exposes the admin service on a dedicated port (default: `6651`). All admin commands
accept an `--admin-address` flag to specify the coordinator's admin endpoint.

```shell
$ oxia admin --admin-address localhost:6651 <resource> <subcommand>
```

## Listing namespaces

List all namespaces configured in the cluster.

```shell
$ oxia admin namespace get
$ oxia admin namespace get my-namespace
```

## Managing namespaces

Create, patch, or delete namespaces.

```shell
$ oxia admin namespace create my-namespace --initial-shards 16 --replication-factor 3
$ oxia admin namespace patch my-namespace --replication-factor 5
$ oxia admin namespace delete my-namespace
```

## Managing data servers

List, create, patch, or delete data server entries. Data server labels are specified as
`key=value` pairs and can be repeated.

```shell
$ oxia admin dataserver get
$ oxia admin dataserver get server-1
$ oxia admin dataserver create server-1 --public server-1.example.com:6648 --internal server-1.example.com:6649
$ oxia admin dataserver patch server-1 --label zone=us-east-1a
$ oxia admin dataserver delete server-1
```

## Splitting a shard

Split an existing shard into two child shards. This operation redistributes the hash range of the
original shard between the two new shards.

By default, the shard is split at the midpoint of its hash range. You can optionally specify an
explicit split point.

```shell
# Split shard 0 at the midpoint
$ oxia admin shard split --shard 0

# Split shard 0 in a specific namespace
$ oxia admin shard split --namespace my-namespace --shard 0

# Split at an explicit hash point
$ oxia admin shard split --shard 0 --split-point 2147483648
```

### Split shard flags

| Flag              | Description                                              | Default     |
|-------------------|----------------------------------------------------------|-------------|
| `--namespace`     | Namespace of the shard to split                          | `"default"` |
| `--shard`         | Shard ID to split (required)                             |             |
| `--split-point`   | Explicit hash split point (defaults to midpoint of range)|             |

After a successful split, the command returns the IDs of the two new child shards.
